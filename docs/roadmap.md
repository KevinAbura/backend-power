# Roadmap

## V1

Backend Power V1 focuses on a repository-grounded backend delivery workflow:

- [ ] Figma design reference model
- [ ] GitHub Issue adapter
- [ ] design-to-work-item extraction
- [ ] traceability persistence model
- [ ] code similarity retrieval
- [ ] pattern context builder
- [ ] patch-oriented generation flow
- [ ] test execution hooks
- [ ] pull request creation
- [ ] review checklist engine

## Later

Potential next steps after V1:

- [ ] Jira integration
- [ ] richer repository indexing
- [ ] staging verification workflows
- [ ] issue lifecycle sync
- [ ] release evidence collection
- [ ] policy-driven action gates

## Guiding constraints

Backend Power should continue to follow these rules as it grows:

- prefer repository-native patterns
- avoid unnecessary abstraction
- preserve traceability from design to implementation
- keep critical actions reviewable
- optimize for real team adoption rather than fully autonomous demos
