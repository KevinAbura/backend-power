# Contributing

Thanks for your interest in Backend Power.

## Project focus

Backend Power explores repository-grounded backend delivery workflows.

Please keep contributions aligned with these principles:
- traceability first
- prefer extension over invention
- repository-grounded generation
- patch-oriented implementation
- human review stays in the loop

## Good first contribution areas

- improve design ingestion structure
- refine work item planning prompts
- enhance code similarity retrieval
- improve review checklists
- expand documentation and examples

## Pull request expectations

Before opening a PR, please make sure:
- the scope is clearly described
- related issue or design context is linked when applicable
- changes follow existing project structure and naming
- unnecessary abstractions are avoided
- documentation is updated when needed

## Discussions

Architecture discussions, trade-off discussions, and implementation ideas are welcome.
