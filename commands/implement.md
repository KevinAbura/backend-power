# bp:implement

## Purpose

Generate a patch-oriented implementation plan or code change using repository-native patterns.

## Inputs

- approved issue
- reference implementation set
- implementation constraints

## Outputs

- implementation plan
- proposed patch
- files to change
- tests to add or update

## Usage

Run this command only after reference retrieval has completed.

## Notes

The goal is not greenfield generation. The goal is to extend current modules and conventions with the smallest sensible patch.
